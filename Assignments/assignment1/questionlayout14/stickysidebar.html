Section 1 — Introduction

This demo shows a sticky sidebar on the left and a fixed box at the bottom-right for comparison. Scroll this main area — the sidebar will remain visible (sticky) while the main content scrolls.

Sticky positioning keeps the element within the normal flow of the document and it only becomes fixed once a defined threshold (top) is crossed.

Section 2 — How sticky works

position: sticky acts like position: relative until the viewport reaches the offset (e.g. top:16px); then it behaves like position: fixed but only within its parent.

Because it's bounded by the container, it won't overlap footer content if the container ends before the page.

Section 3 — Fixed vs Sticky

position: fixed (see the dark box at bottom-right) is attached to the viewport. It remains visible no matter where you scroll and does not influence or get influenced by the layout flow.

Use fixed for global UI controls (e.g., chat widget, back-to-top button). Use sticky when you want an element anchored while still behaving like part of the layout.

Section 4 — Practical uses

  • Table of contents that follows the reader.
  • Sticky headers that remain visible while reading long articles.
  • Sidebar filters on e-commerce pages.

Section 5 — Try it out

Resize the browser and scroll. On narrow screens the layout stacks and the sidebar appears above the content. The small-screen behavior ensures usability on phones and tablets.
